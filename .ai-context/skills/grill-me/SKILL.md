# grill-me

実装や大きなリファクタの前に、**ユーザーと仕様を完全一致**させる。

## 手順

1. **目的・スコープ** — 何を達成するか、何をやらないか（1–2 問）
2. **仕様** — 入出力、API、UI、データモデル（2–3 問）
3. **エッジケース** — 失敗、権限、空状態、同時実行（1–2 問）
4. **運用** — テスト、マイグレーション、デプロイ、ロールバック（1 問）

## ルール

- 一度に **1–2 問** まで。選択肢がある場合は AskQuestion を使う。
- 推測で埋めない。不明点は必ず確認。
- 回答を受けてから次の段階へ。スキップしない。
- 合意後に Plan または TODO リストを提示し、実装開始前にユーザー確認を取る。
